用户体验指标体系

发表于 2020-07-23 分类于 资料收集

前言

假如我还想继续投递用户研究的相关岗位，那么我必须回答一个笔面试常见问题：建立一个用户指标体系及常见用户体验指标。类比于设计问卷的概念操作化，建立用户指标体系虽同样是概念分解、指标建立，但在这里永远只有一个概念，还有一些固定的指标。

固定指标来源于APP、网站的所收集用户行为，在程序上设定，程序员的设定就是继承常见指标，基本是一样的；用户体验的概念则会随具体产品、产品目标而变化。

本文目的在于，了解常见用户体验指标，即程序上的常见设定；了解一般用户体验概念，从而在面对具体产品时有所准备，并在此基础上变化。

Measuring the User Experience on a Large Scale: User-Centered Metrics for Web Applications

Ro翻译的Google文章

原文2010年发布，的确是很老的文章了

更全的翻译

传统网站指标 PULSE

基于商业、技术的产品评估，重要有效，但指标层次太浅、不直接与用户体验相关、有误导

  • page view 页面访问次数
  • uptime 持续运行时间
  • latency 延迟
  • seven days active user 7天活跃用户数
  • earning 收入
HEART框架

基于PULSE存在问题，提出的补充框架

  • Happiness 愉悦感
  • Engagement 参与度
  • Adoption 接受度
  • Retention 留存率
  • Task success 任务完成度
关于使用
  • HEART为衡量范畴，不同产品可由此定义特定指标，以监控目标达成
    • 愉悦感：用户体验中的主观问题，如满意度、视觉感受、向人推荐意愿、易用性等，通过问卷长时间监控相同指标以监测前后变化
    • 参与度：用户在产品中的参与深度，一段时期内访问频率、强度、互动深度，如单用户每周访问次数、用户每日上传图片数量（用单用户而不是总量，避免用户增长而导致的总量增长）
      • ==参与即互动==，可参照互动概念。互动有广度、深度，对比到APP使用，用户每日使用时长（今天竞争的关键就在于争取用户时间）、每日访问频率（前者是深度，这个大概就是广度）等等
    • 接受度、留存率：特定时期内的大量用户统计，理解新老用户问题相关
      • 接受度：特定时期内多少新用户开始使用产品，如7天內新增账号
      • 留存率：活跃用户留存率，7天活跃用户在下个周期的留存率
      • ==如果==说满意度、参与度，是跟用户的体验很相关的；留存率直观显示用户使用后体验，喜欢、有用就留下；接受度显示产品对用户的吸引力；但存在怀疑，==用户中心还是产品中心？==
    • 任务完成度: 效率、效果、错误率
  • 不需要在一个指标体系中用到所有纬度，可根据情况包括或排除某维度
目标—信号—指标 流程

阐明产品、功能目标，定义达成信号，建立合适指标

  • 目标
    • 产品、功能的目标是什么？用户完成什么任务？新设计试图达到什么？
    • 使用HEART框架提示相关目标，如吸引新用户更重要还是鼓励现有用户重要
    • 无需担心找不到相关信号、指标
  • 信号
    • 用户的行为、态度如何体现成功与失败？什么行动表示目标已达到？
    • 思考信号的可能数据源，日志、问卷
    • 选择与敏感的、与目标特别相关的信号
    • 失败比成功更容易定义
  • 指标
    • 思考信号能否转化为指标，以便持续跟踪
    • 原始统计数据随用户数增长，比例或用户平均值更为有用
    • 确保精度，如过滤爬虫、垃圾信息的流量
    • 需要与其他产品对比时，增加监控指标
评价

此方法成文于2010年，当年还是网站的时代，而非移动互联。HEART模型现在看也有效。简单检索就可以找到好几个这篇文章的链接，此文热度不消，是此领域的经典文献。但对于了解用户体验指标体系而言还不够。应补充一些APP方面的相关新变化。

Pamela Pavliscak

sketering的翻译

  • 用户体验指标跟踪产品体验变化，并为设计提供参考
  • 但如转化率、用户参与时间这些指标层次过高、抽象，无法为设计提供参考（因果关系复杂，无法区分是什么导致的）
现有问题
  • 容易被跟踪、监控到的指标并不意味真正重要，不能代表体验如何
    • PV对广告是重要衡量指标，但不能表明用户在什么时间段使用APP、使用后体验如何
      • PV是网站时代网站的广告销售重要指标，过去时代的遗产，非当前的核心
  • 指标含糊、不清晰
    • 用户在网站、APP上的停留时常不能清晰界定用户参与度，可能为困扰时间
      • 此处同样是无法了解用户停留时间到底干什么了
  • 指标并不总是映射到设计上
  • 跟踪了太多指标，而正确的指标根本没有抓取
    • 大数据能够获得什么洞察？或者在大数据下，算法更加关键。而算法代表着思维，思维才能洞察
    • 当前的用户体验指标体系至少有一部分是其他团队的KPI（即业务导向的指标），缺少用户体验设计的相关指标，无法给设计提供建议（这也是指标不能反映到设计上的原因）
      • 或许可以这样理解：广义的用户体验指标体系包含了商业指标、用户体验指标、用户体验设计指标
